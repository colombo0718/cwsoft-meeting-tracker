# 森昱 IMEI / SIM 庫存對帳全面普查

## 任務

針對森昱資料庫中「會有 IMEI / 序號的類別」做全面普查，範圍包含：

- 類別編號 `0`：行動電話
- 類別編號 `2`：易付、儲值卡
- 類別編號 `3`：SIM卡

其中 `dbo.庫存IMEI` 內部查序號資料時使用 `類別編號 in (0,2,3)`，與 `商品類別` 表查得的結果一致，因此本次對帳以 `0,2,3` 為準。

### 對帳公式

`算數在庫` 定義為：

```text
進貨 + 銷貨退回 + 入庫 + 轉換進 + 出貨退回
- 銷貨 - 進貨退出 - 出貨 - 出庫 - 轉換出
```

`差異` 定義為：

```text
庫存即時用.SUM(店內庫存量) - 算數在庫
```

### 單據計算口徑

本次彙總時，各單據主檔均排除 `Invalid = 1`：

- 進貨單
- 銷貨單
- 銷貨退回
- 進貨退出
- 出貨單
- 出貨退回
- 入庫單
- 出庫單
- 轉換單

### 未納入本次主公式的型別

以下型別未納入本次十種單據公式，但已額外查證對 `SHA329312805` 沒有影響：

- 維修物品：0 筆
- 預約物品：0 筆
- 庫存盤點記錄：0 筆
- 庫存還原記錄：0 筆
- 借入 / 借出 / 還入 / 還出：`商品編號` 為 `int`，不適用 `SHA...` 這類字串商品編號
