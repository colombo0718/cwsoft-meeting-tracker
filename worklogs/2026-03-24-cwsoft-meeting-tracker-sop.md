## 討論摘要

### 1. 問題背景

雅婷語音辨識逐字稿準確度不穩定，需要人工校正語者名稱、客戶名稱、專案名稱，且過去 11 份歷史逐字稿（2026/1~3 月）尚未整理。

### 2. SOP 建立

**決策**：建立標準化會議紀錄整理模板，包含：
- **摘要**：本次會議核心結論
- **決議**：明確的行動決策
- **未定事項**：待討論或待確認的議題
- **待辦清單**：具體行動項目與負責人

**校正規則**：制定語者、客戶、專案名稱的自動/人工校正規範，解決辨識錯誤問題。

### 3. 批次處理歷史逐字稿

一次性處理 11 份歷史逐字稿，並根據會議內容初始化 9 個專案的狀態追蹤檔（包含優惠券系統、小葳 AI 等）。

### 4. GitHub 儲存庫與看板

建立 `cwsoft-meeting-tracker` GitHub repo，以 README 作為專案看板，顯示各專案當前狀態。
