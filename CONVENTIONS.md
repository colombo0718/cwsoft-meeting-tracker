# CONVENTIONS.md — worklog 寫作規則

> 規範「人對 AI 工作紀錄」（`worklogs/`）的寫法。
> 與「處理一份新逐字稿」SOP（見 `PROJECT.md`）平行存在、兩者性質不同：
>
> - 逐字稿是把人類會議結構化（雅婷語音 → 標準會議紀錄）
> - worklog 是把 AI 協作脈絡留存（cwsoft 同仁 × 各種 agent 推進工作的紀錄）

---

## 何時寫 worklog（AI 主動提醒的 Trigger）

當以下任一條件成立，AI 在回覆末尾主動提醒 colombo：

> 「這段討論建議整理成 worklog，要我幫你寫嗎？」

- 對話超過約 30 則來回
- 完成了一個架構決策或技術選型
- 有實質改檔或建檔（非閒聊）
- 待辦或設計方向有重大變動
- 涉及多個 CWSoft 子專案的跨界討論
- 確認了某個 CWSoft 工具的新做法

不確定要不要寫時、寧可問、不要漏。**漏記的成本 > 多問的成本**。

---

## 檔案命名

```
worklogs/YYYY-MM-DD-主題關鍵字.md
```

- 日期為討論**實際發生日**（非整理日）
- 主題關鍵字用 kebab-case，2-4 個英文詞
- 同天多場加 `-2`、`-3` 後綴

---

## Header 格式（四欄位必填）

```markdown
# 標題（這次討論聚焦的主題）

- 日期：YYYY-MM-DD
- 主機：筆電 / 公司主機 / 個人主機
- 參與者：colombo0718 × Claude (model name)
- 相關專案：general-task-bot、digital-agent-xiaohao
```

### 欄位說明

**「日期」**
- 填**討論發生時**的日期
- 若整理日與討論日不同，下方加一行：`（整理日期：YYYY-MM-DD）`

**「主機」**

| 填寫值 | 指的是 |
|--------|--------|
| `筆電` | 主力開發機，創業 + 學業主要在這 |
| `公司主機` | cwsoft 工作機 |
| `個人主機` | 常駐服務主機 |

跨多台機器操作填 `筆電 + 個人主機` 等多值。

**「參與者」**
- 主要格式：`{使用者} × {agent} (model name)`
- 例如：
  - `colombo0718 × Claude (claude-haiku-4-5)`
  - `彥偉 × Codex (gpt-5.4)`
  - `羿宏 × Cursor (claude-sonnet-4-6)`
- 使用者填 cwsoft 同仁的稱呼（自己的名字 / 帳號）
- agent 填工具名 + 模型名

**「相關專案」**
- 列本輪**真正有關**的 CWSoft 子專案
- 多個用頓號分隔：`general-task-bot、cwsoft-liff-pages`
- 純流程性 / 不歸特定專案：填 `跨專案`

## 必要段落

每份 worklog 至少包含以下三個區塊：

### 1. 討論摘要

以有編號的小節（`### 1. 主題` / `### 2. ...`）呈現各個議題。

每個小節包含：
- **背景 / 問題**（可選，若不言自明可略）
- **決策** 或 **結論**（黑體標示）
- 理由或補充說明

> **替代格式（允許）**：當議題有天然大類分組時，可用 `## 一、二、三` 大節取代三段式，但最後一節仍必須是 `## 待跟進`。

### 2. 本輪新增 / 更新檔案（可選）

若有實際修改或新建檔案，列出清單，標明所屬子專案。

### 3. 待跟進

- 標題統一寫 `## 待跟進`，不用其他別名（「待辦」、「下一步」等）
- 用 `- [ ]` 條目列出待辦，不加截止日（追蹤交給 `projects/{專案}.md`）
- 已完成的項目改為 `- [x]`，不要刪除

---

## 整理原則

- **決策明確**：每個討論點最後要寫清楚「決策是什麼」，不只是「討論了什麼」
- **關聯清楚**：`相關專案` 要列本輪真正有關的，不要全列
- **精簡優先**：摘要不是逐字稿，聚焦關鍵決策與背後理由
- **不重複記**：若決策已寫進 `projects/{專案}.md`，這裡只留摘要不全文複製
- **守邊界**：worklog 不寫 LL 宇宙或 colombo 個人創業相關內容（見 `PROJECT.md` 的治理層定位段）

## 與 MM 的 CONVENTIONS.md 對應

本檔結構參考 `matrix-manager/CONVENTIONS.md`（LL 宇宙的會議紀錄整理規則），
改寫成 CWSoft 語境的 worklog 版本。

差異：
- LL 版的「相關專案」列 LL 子專案；本檔的「相關專案」列 CWSoft 子專案
- LL 版適用「colombo × Claude 的會議紀錄」；本檔適用「cwsoft 同仁 × 各種 agent 的工作紀錄」
- 本檔多一條「守邊界」（不洩漏 LL 內容）
- 本檔的使用者不限 colombo——任何 cwsoft 同仁都可以寫 worklog
