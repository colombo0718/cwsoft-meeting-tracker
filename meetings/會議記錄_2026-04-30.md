## 3) 討論摘要

### 主題1｜折價券/振興券是「付款方式」（澄清）
- 銷售單上的「明鏡/筆卷」欄位 = 各種付款方式（不是折扣）
- 例：總額 2,000，現金 1,000、折價券 500、分期 500
- 之前發過：振興券、挪娃國券、現金券
- 25 年前到現在，門市發折價券概念又回來了
- 結論：折價券**不適合做成商品折扣**，本質是付款方式

### 主題2｜銷售單下方「點數」欄位是給誰用的？
- 這欄位是早期為了**遠傳/台哥大**設計的（不是給商品點數用）
- 規則極其複雜：
  - 月租升轉/續約升轉/降轉 → 點數不同
  - 低於某金額不計點
  - 上限蓋頂、底蓋
  - 整套公式算出每月分數
  - **規則一直在變**
- 結論：這欄位之後再做「結論性」整合，不重要、優先級低

### 主題3｜報價單需有有效期 + 線上繳費機制
- **痛點：** 之前口頭報 9,000 沒押日期，客戶事後不認賬
- **規則：**
  - 月租類報價：有效期到當月月底
  - 特殊專案類報價：一週
  - 過期不認
- **未來方向：** 報價單做成「直接繳費」（會款後給後五碼）
  - 減少請款人力
  - 進階行銷、新功能都要走這流程
- **彥偉重要原則：** 「**價格可以減，服務不能少**」（為了第二年續約）

### 主題4｜零壹（靈異）試用流程與 DOMAIN 處理
- **時程拉長：** 6/1 才開始計費（送一個月試用），中間測試期不算錢
  - 原因：5/9 士豪要出國一週，這段不能撐 cover
  - 「我比較喜歡把時間拉寬」（彥偉）
- **DOMAIN 問題：**
  - 客戶有現有官網 DOMAIN，不歸我們管
  - 解法 A：客戶工程師把 DOMAIN 指過來我們的 IP
  - 解法 B：我們申請新 DOMAIN（免費），用 Retarget 過去（不顯示我們網址）
  - 結論：先幫客戶申請我們這邊新 DOMAIN，無所謂，不用錢；客戶有意見再談
- **接下來步驟：**
  1. 開設定畫面 + 匯會員資料
  2. 在測試庫先把資料試一次（看有沒有問題）
  3. 開始測試
  4. 5/9–15 士豪出國，期間業務「換時間」說：「負責這塊的人出國度假」
  5. 6/1 完整上線

### 主題5｜模板化（第一個客戶教訓）
- 彥偉之前做第一個客戶時沒想到要做模板，後面開第二、三家都很累
- **要求：** 零壹這次要把模板（空的資料殼）建起來，下次直接複製
- 「沒想到會大賣啊」（彥偉笑談）

### 主題6｜進階行銷 LIFF 主選單：四個 vs 六個
- 零壹要四個選單（我的點數、我的優惠等）
- 彥偉建議**六個**較好看，且預留空間
  - 雨心（宇新？）一定會變六個（要加維修單進度）
  - 不過彥偉自評「四個 vs 六個」像是「打腫臉充胖子」（內容不夠）
- **結論：** 為靈異做四個（按客戶實際），未來內容多時可選

### 主題7｜專案點數計算（重要邏輯）
- **問題：** 銷售單前面是商品（保險）、後面是專案，兩個都有點數會重複算
- **解法 A：保險類別**
  - 建一個「保險」商品類別
  - 把所有「保險」字樣商品歸入此類別
  - 賣保險類別自動給點，前面商品計就好，後面專案 skip
- **解法 B（最簡單）：在專案上加「可記點 / 不可記點」checkbox**
  - 客戶自己勾，可記點才算
  - 一刀切，最簡單
- **目前專案欄位：**
  - 已有「新辦 / 續約 / NP」combo
  - 已有「是否恩批」勾選
  - **歷史包袱：** 20 年前只有新辦/續約 → 後來加 NP（另開欄位）→ 又加「異動服務」（補卡、加值服務）
- **要問靈異規則：**
  - 新辦才算 / 續約也算 / NP 也算？
  - 哪些情況不算？
  - 「保險類前面算就好，後面專案不算」是否能做？
- **客戶層配置：** 邏輯層拉出來，每客戶可有自己規則；基本款八點公版

### 主題8｜維修平板版（送一年綁約策略）
- **客戶（雨心？）需求：** 維修師不想離開維修台跑去 POS 點來點去
- **解法：iPad 版維修系統**
  - 維修師只在平板上點選維修狀態（修到哪、用什麼料）
  - 跟 POS 資料連動（網頁形式即可）
  - 像「叫車系統」那樣的進度狀態 UI
- **業務策略：**
  - 一台原價 250 元/月
  - **「送一年」** 給客戶綁約（必須先繳進階行銷年費）
  - 一年後開始收 → 客戶習慣後不會為了 250 元退掉
  - 「叫飲料都超過一千塊，不會為這個省」
  - 老闆愛這套：先讓他流口水 → 屌住再收
- **demo 計畫：** 下次帶 iPad 去 demo

### 主題9｜小幫手鬼打牆（點數語意）
- 士豪：問點數時小幫手會鬼打牆
- 原因：「點數」有多種：簡訊點數、月租點數、會員點數
- **解法：** 不明確時必須反問「請問你講的是月租點數還是簡訊點數？」
- 設計信心值機制 + 歷史紀錄學習
- 「結束此回合」要加 Yes/No 確認機制（防止使用者中途想換問題）
- 開後門 + 腳本灌測試訊息（已在 4/27 提）

### 主題10｜2025 backup 清理 + 自動轉檔
- 士豪清掉 2025 年備份，老闆說「早就該清了」
- 自動清理機制建議：
  - 凌晨 3–4 點 IDLE work
  - 容量低於 X、月份高於 6 個月（或 4 個月）即清
  - 「除非你拿來當樂趣，不然叫他自己做就好」
- 副流程：羿宏寫自動轉換腳本（轉檔），士豪那邊有備份；要對寫
- 士豪論文：實驗角昨天做完，下週開會，再一個月口試

### 主題11｜AI 助理（POS AI）長遠規劃
- **未來方向：用講的時代**（彥偉想買翻蓋手機貼車上遠端用）
- **架構：** Codex 透過 CLI 接收訊息 → 給 Source Code → 定義 CLI 動作（Open 單據、Find 單號）
- **降低 token 策略：**
  - 預先把每張單的邏輯寫成 MD 檔，AI 直接讀（不必每次重新分析 source）
  - 智能判斷用聰明模型，執行用最笨模型（如 Gemini 3.5）
  - Compile 用 Python 包 VB（VB 不能直接執行）
- **AI 助理商業化（後期）：**
  - 「Post AI 助理」客戶可額外付費購買
  - 用講的問：「這專案多少錢？這個毛利多少？」
  - 短期不開放給客戶，先內部測試穩定

### 主題12｜彥偉業務策略總結
- 「**做第一家要非常非常小心**，不要把招牌砸爛」
- 第一個客戶 6 間店開到 100+ 間 → 廣告效益巨大（協訊同行 2003.12.6 簽約）
- 服務做好是為了客戶幫我們宣傳
- 「**為你量身訂做一套產品**」勝過標準化標單
