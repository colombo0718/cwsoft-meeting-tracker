## 3) 討論摘要（依主題）

### 主題1｜LINE OTP 會員綁定技術實作討論
- 現況/背景：進階行銷系統的 LINE OTP 會員綁定功能是目前最優先的開發項目，核心目標是在客戶完成手機號碼 OTP 驗證後，將 LINE 帳號與 OA（官方帳號）下的 user ID 正確綁定，以便後續精準推播。
- 核心討論：
  - 技術難點：從網頁端直接抓取 LINE user ID 有技術限制，士豪提出的折衷方案：在 OTP 驗證按下的同時，從 LINE 那端傳送一則確認訊息（如「OTP 認證成功」）到用戶的 LINE，藉此取得 user ID。
  - 羿宏說明：只要訊息是傳入特定 OA，系統就能知道是哪個 user ID 發送的，因此可以從接收方確認 user ID，技術上可行。
  - 羿宏類比銀行 OTP 邏輯：網銀的裝置 OTP 是在裝置已完成綁定後才會傳送，這裡的邏輯也類似——OTP 驗證完成之前不發任何訊息到 LINE，驗證完成後才發一則「認證完成」通知。
  - 設計原則：驗證完成後只傳送「認證完成」的靜默訊息，不發多餘內容，確保不干擾用戶體驗。
- 產出/結論：技術路徑可行，以「OTP 驗證完成後觸發 LINE 訊息傳送取得 user ID」作為實作方向。目標為驗證流程中不出現多餘訊息，並確認裝置對應 user ID 後完成綁定。

---

## 4) 決議
- LINE OTP 綁定實作方向確立：OTP 驗證完成後觸發 LINE 端訊息，取得 user ID 完成綁定。
- 驗證過程中不提前傳送任何 LINE 訊息，確保只在認證完成後才通知。
