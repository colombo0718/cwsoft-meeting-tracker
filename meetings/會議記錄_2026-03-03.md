## 3) 討論摘要（依主題）

### 主題1｜宇新維修加盟中控系統提案投影片逐頁審查
- 現況/背景：羿宏使用 LLM（語言模型）生成投影片初稿，本次三人共同審查內容並逐頁修改。
- 核心討論：
  - 投影片由 AI 生成，部分圖片與主題不符（如出現飯店相關圖）或無法辨識（如一堆齒輪的圖），需更換為維修相關圖示。
  - 市場現況頁面：維修與二手機回收是一體兩面，二手機回收可以和維修結合成完整服務鏈（修好再賣）。
  - LINE 整合方向：
    - 維修進度通知（收件 → 排程中 → 檢測中 → 維修中 → 更換料件 → 完成）讓客戶透過 LINE 查詢，降低電話詢問量。
    - 客戶手機在維修時可能無法收到 LINE，現場等候為主要情境。
    - 對散客可透過 LINE 導引加入會員，通訊門市客戶可透過 POS 串接直接轉發維修單。
  - 官網入口：宇新已有自己的網站，無需重建，改為在 LINE 圖文選單左上角加入「官網入口」連結到他們現有網站即可。
  - SEO 部分：AI 興起後傳統 SEO 效益下降，建議投影片改為「官網與 LINE 官方帳號連結」，不強調 SEO。
  - 留客引擎：會員資料庫 + 點數規則，是進階行銷系統的核心賣點。
  - 維修加盟中控系統：各加盟門市維修單集中到宇新總部的 Dashboard（儀表板），依維修師/分店顯示維修數量與進度。
  - 投影片修改方向：
    - 移除不相關圖片，換成維修相關圖示。
    - 「市場現況」改為白話描述，去掉 SEO 字樣改為「官網入口連結」。
    - 官方帳號觸達維修站：補充可透過 POS 系統連動，或直接透過 LINE 官方帳號管理。
    - 生態鏈構想補充頁：如其他門市用我方 POS，可直接串接轉送維修單，零一等客戶也可轉介。
  - 彥偉提出生態鏈概念：若其他客戶（如零壹通訊行）也使用我方 POS，可直接在系統間轉送維修單，形成生態系，我方從中獲得更多廣告價值。
  - 維修加盟中控系統名稱討論：羿宏建議改名為更直覺的說法，不用「中控」，改為「維修加盟管理系統」。
- 產出/結論（暫定）：投影片修改清單確認，下次拜訪前完成修訂版。

### 主題2｜維修站業態分析與 LINE 整合
- 現況/背景：討論維修站整合 LINE 官方帳號的可行方式。
- 核心討論：
  - 宇新的加盟店目前有 LINE 維修群，但群內帳號不是官方帳號。
  - 建議讓各加盟門市申請自己的 LINE 官方帳號，若無法，也可透過 POS 系統連動通知。
  - 兩種通知方式：① 若有 POS 系統串接 → 透過 POS 事件觸發 LINE 通知；② 若無 POS → 直接透過 LINE 官方帳號管理。
  - 風華案例分析：風華是單一老闆自修，不擴大，已介紹我方 POS 給其他門市；宇新是想擴大加盟，兩者需求不同。
  - 風華目前需求：維修報表（月統計），類似消費明細，功能需求簡單。
- 產出/結論：維修進度通知流程確認加入投影片，LINE 整合方案提供雙選（POS 串接或獨立 LINE 管理）。

### 主題3｜優惠券系統設計方向
- 現況/背景：宇新提案包含優惠券功能，本次討論如何在投影片中呈現，以及實作設計方向。
- 核心討論：
  - 彥偉建議：自行開發優惠券系統（而非使用 LINE 原生優惠券），好處是可與 POS 整合、顯示在 LIF 網頁、做到核銷控制。
  - LINE 原生優惠券適合小店（無 POS），大廠都是自建優惠券系統整合 POS。
  - 優惠券類型建議（投影片補充）：買一送一券、折價券、生日優惠券、介紹獎勵券等。
  - 優惠券可透過 QR Code 方式核銷，確保不被盜用。
  - 生日券發送邏輯：每月自動發送下個月生日的會員，有效期為生日當月。
  - 研究方向：參考蝦皮消費券的操作模式，整理優惠券類型後加入投影片。
- 產出/結論（暫定）：士豪研究 LINE 優惠券 API 開放程度；投影片加入優惠券類型說明頁。
