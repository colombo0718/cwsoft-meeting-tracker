## 3) 討論摘要

### 主題1｜進階行銷 demo 準備
- **點數規則設定**（POS V3 測試論點）：
  - 行動電話類、中古機、手機配件分別設定點數
  - 無卡分期不算（抓銀行的）
  - 中古機用「三比平」(三比評) 自動判斷二手機
- **點數兌換頁面**：把驗證頁面砍到剩「驗證中」，不要顯示一堆人名（沒必要）
- 禮拜三 demo 前要把畫面修好

### 主題2｜點數扣除三情境（重要邏輯）
| 情境 | 走的流程 |
|------|---------|
| 銷售退回 | 扣點 |
| 單據刪除 | 走「**點數回數**」（不走銷售退回） |
| 單據修改 | 重新計算成正確值 |

### 主題3｜點數運動紀錄 vs 會員資料書 — 介面爭論
- **羿宏方案：** 在「平台發點」做單獨頁面（左：查詢、中：發點、右：運動紀錄）
- **彥偉反對：** 跟「會員資料輸入」功能太重複，初學者會搞不清楚要進哪個
- **結論：**
  - 歷史點數紀錄做成「另一個按鈕」，從會員資料書點過去（同一介面、同一畫面）
  - 沒買進階行銷的客戶 → 點到時跳介紹影片
  - 群發功能可保留（跟單發是不同需求）

### 主題4｜進階行銷模組分群與 pop-up 介紹
- **模組拆三塊**（DLC/extension 概念）：
  1. 會員 extension（增強）
  2. 點數等級
  3. LINE 整合
- **pop-up 介紹更新：**
  - 之前 auto answer 已過時（「剛寫完就沒用了，AI 進化比我快」— 士豪）
  - 要更新成現有功能清單，最好做一支介紹影片
  - 至少要條列式列出有什麼功能，可以丟給客戶看
- **業務感受：** 一個項目變六個項目，看起來「東西多」，賣得動

### 主題5｜點數 ↔ 優惠券聯動設計
- **流程：** 點數 → 換優惠券 → 優惠券折扣
- **跟 Ocard 差異：**
  - Ocard 是兩套不相干系統（Ocard 在手機消券，POS 直接折 500 元）
  - **CWSoft 要做整合**（這是優勢、是賣點，不要兩套分開）
- **要做的：** 點券歷史紀錄可查；點數 ↔ 優惠券「溝通流程」要建立
- 各家客戶的優惠券使用規則（滿千才用 / 折抵金額）由優惠券系統管，點數系統只管「換什麼券」

### 主題6｜義峰客戶 — 繳費卻跳停用警告（最高優先級）
- **症狀：** 義峰已付錢、有 12 點，但 POS 還是跳「線上付款停用倒數」警告
- **根本原因：** 彥偉之前在 4 月手動改過義峰的「線上付款停用時間」 → 避開了系統自動更新機制
  - 自動更新機制：每月 1 號凌晨 by SQL（看點數還夠不夠 → 往後延）
  - 義峰被改掉後，自動更新沒再去動他
- **點數扣點 SQL 程式還沒寫**（上線 4 個月仍未完成）
- **副問題：** 義峰有三家分店「義風網路」「寶山南華」「連強」，實際只有南華店在用
  - 12 點 ÷ 3 店 → 4 月就用完
  - 要把另外兩店關掉
- **關閉分店的步驟（FK 約束）：**
  1. 先確認該店完全沒銷貨/進貨/異動
  2. 清庫存（庫存非零會擋住）
  3. 關閉分店
- **彥偉重要原則：**
  - 「**有付錢不能用 = 第一優先**」（比其他什麼都重要）
  - 「**收頭不收尾**」：先付一年的錢才給用，不接受後付（倒帳追不回）
  - 老闆跑路：聯絡分店長/管理局，誰承接誰付錢，不認賬

### 主題7｜AI 客服 / 小幫手強化
- **Bug 修正：** 群組訊息接收（之前只有單對單）、圖片支援、信心值機制
- **信心值機制：**
  - 目前抓 70 分太低 → 彥偉建議 **80–90** 才不問
  - 信心值不夠 → 觸發澄清機制（按鈕反問 / 等使用者再講）
  - 顯示文字要清楚：`at_just_point` 等內部代碼要改成「月租點數」等人類可讀
- **訊息流水紀錄三層：**
  - 流水紀錄（in/out 訊息）
  - 提問→等回答的記錄
  - 萃取值 → API 處理結果
- **看物表（mapping）機制：**
  - 確認過的對應關係 → 下次先查 mapping correction 再丟 LLM
- **辨識誤判案例：** 「義鋒幾點」→ 變成「Name 義風」（疑似 AI 動態改 code 導致）
- **降噪需求：** 開車環境噪音 60–70 dB 講不清楚 → 要在 Leaf 網頁端做人聲降噪
- **多輪對話：** Session ID 機制保留（一個 session 多輪）
- **腳本灌測試訊息：** 開後門讓腳本模擬 LINE OA 訊息，不必每次都靠真 LINE 訊息測試
- **LLM log 切割能力：** error message 黏在指令前後，LLM 能自動切開判斷
- **未來方向：** 手機端 agent 遙控（彥偉想買翻蓋手機貼車上，遠端 run agent）

### 主題8｜報價策略
- **價格：** 主系統 3,000/月 + 每分店 1,000/月
  - 例：四家店 = 3,000 + 3 × 1,000 = 6,000/月
- **比較：** Ocard 是 3,000 同一官方帳號（沒分店加價）
- **CWSoft 優勢：** 與 POS 整合（功能更全、跟 POS 共用會員/銷售資料）
- **合約：** 第一年特價，第二年回原價（要寫進合約）
- **重要原則：「要買就全買」**
  - 不接受客戶只買部分分店
  - 否則點數系統設計會卡（一店有買一店沒買，點數能不能跨店？）

### 主題9｜門市導覽自動化（demo 用）
- 目前：圖片要手動換
- 目標：從分店地址（分店資料書已有）→ 自動 Google 抓圖
- 切換不同客戶 demo 時可一鍵換
- 沒抓到圖時讓客戶上傳

### 主題10｜NotebookLM / 投影片產生
- 教學影片用 NotebookLM 自動產生（之前實驗教學在用）
- **客戶介紹投影片要精簡：** 5–10 頁、每頁一個重點、針對痛點
- 業務上投影片不需太多花俏內容，明確的點到就好

### 主題11｜AI 工具雜談
