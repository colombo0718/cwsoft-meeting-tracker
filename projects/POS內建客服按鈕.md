# 專案：POS 內建客服按鈕（「我有問題」按鈕）

## 當前狀態
> 規劃階段。2026-05-12 老闆會議拍板（會議記錄_2026-05-12 主題9），由羿宏負責。2026-05-14 colombo 與 AI 整理出資料打包設計與動工計畫。

## 動機
- 客戶截圖截不到重點、回報「版本最新」實際上不是
- 客服每次要追問「你哪一版？」「哪個畫面？」「錯誤訊息是什麼？」「印表機型號？」浪費大量時間
- 老闆 2026 年底目標：10% 客服自動處理；按下按鈕自動採集環境資訊是基礎建設

## 目標
讓 AI 拿到單一個包就能 90% 自診斷，剩 10% 才需要追問。

## 階段拆分

### 階段 1（MVP，本月可上）
- POS 主畫面加「我有問題」按鈕（常駐）
- 點下去彈一個小視窗，自動填好「必收資料」+ 讓客戶輸入問題描述
- 傳到內部接收端（先簡單一張表存進 DB）

### 階段 2（兩週內）
- 接 LINE OA：客戶按鈕送出後在 LINE OA 收到「我們收到了 案號 #1234」的訊息
- dashboard：客服 / AI 看到完整包

### 階段 3（一個月內）
- 條件收（exception log、Form 截圖）
- AI 串接：包進來自動分析、信心度 ≥95% 直接回，低於就走影子模式

### 階段 4（後續）
- 環境收（印表機列表、Windows locale 等）
- 同類問題自動合併

## 資料打包設計

### 必收（每次都送）
| 項目 | 從哪拿 | 備註 |
|------|--------|------|
| 案號 → 客戶 DB | POS config | 案號用 4-5 碼，POSConfig.分店設定檔 mapping |
| POS 版本（build 時間戳） | 手機進銷存v25.exe FileVersion | 「此版本為 2026/MM/DD HH:mm」那個 |
| 當前 Form 類別名 | `Application.OpenForms` 最上層 | 精確到「設定_專案設定2」這種 class name |
| 時間戳（精確到秒） | DateTime.Now | 對 commit log 用 |
| 員工編號 + 分店編號 | `[global]` 現有變數 | 誰按、在哪間店 |
| Hostname | Environment.MachineName | 區分同店不同電腦 |
| 問題描述（自由 + 引導下拉） | TextBox + ComboBox | 不能全自由，要 dropdown 引導 |

### 條件收（階段 3 加）
| 項目 | 觸發 | 用途 |
|------|------|------|
| 最近 N 條 Exception log | 24h 內有 unhandled exception | 直接拿 stack trace 定位 |
| 當前 Form 截圖 | 永遠收 | 看 UI 狀態 |
| 當前 Form 查詢條件值 | 永遠收 | 看用戶輸入了什麼 |
| 最近 5 條 SQL | POS 有 SQL log 才送 | 對應星威 FindGoodRow 那種 |

### 環境收（階段 4 加）
| 項目 | 觸發 |
|------|------|
| Windows 版本 + locale | 列印 / 時間格式問題 |
| 預設印表機 + 印表機列表 | 列印問題 |
| POS 目錄 .rpt 與 dll 檔案 hash | 安裝損壞懷疑時 |

### 永遠不收
- 整個 DB / 整張表內容
- 會員 PII（姓名、手機、身分證、地址）
- 所有 form 的截圖
- 整個 Windows 事件記錄

## 問題描述引導模板

純自由文字會收到「印不出來」三個字，要 dropdown 引導：

```
問題類型：[ 列印 / 數字異常 / 閃退 / 操作不會 / 其他 ]
發生時機：[ 一打開就 / 按某個按鈕後 / 隨機 / 一直如此 ]
是否影響營業：[ 是 / 否 ]
你看到什麼錯誤訊息？（拍照貼上或打字）：________
```

## 風險與已知坑

| 風險 | 對策 |
|------|------|
| 客戶按按鈕後沒收到「我們收到了」會以為沒送出 → 重複按 | LINE OA 立即回「案號 #1234」收件回執 |
| 1 分鐘按 10 次（會議拍板的閾值）→ 同類問題大量湧入 | 同案號 + 同 Form + 同 exception 在 5 分鐘內合併 |
| 客戶端 LINE OA 沒設定 / 沒加好友 → 走不通 | 第一次按按鈕引導加 LINE OA |
| 敏感資料外洩疑慮 | 截圖前處理（隱藏 PII 欄位），客戶可以預覽要送什麼 |
| 大量上傳影響網路 / FTP 頻寬 | 階段 1 只送文字資料；階段 2 截圖才送，且壓縮 |

## 與其他系統的關係

- **小幫手 / 小葳 AI 客服**：本按鈕是「採集端」，AI 客服是「分析回覆端」。包進來經 AI 分析後走影子模式或自動回。
- **LINE OA**：客戶端唯一回覆介面，所以收件回執走這。
- **POSConfig.分店設定檔**：案號 mapping 來源，可能要擴一欄「案號」。
- **影子模式**：信心度低的就丟進去人工確認。
