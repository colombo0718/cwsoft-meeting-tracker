## 設計備忘
- **集點方式**：計次（各類型固定幾點），非依消費金額
- **商品類型分別設定**：新機、中古機、手機門號、維修…各自設定點數，不綁定
- **點數歷史**：每次存檔時同步寫入（不事後回算），包含：時間、單據號、異動點數、原因
- **維修/二手機**：不在銷貨單流程，另行設計集點入口
- **老闆調整點數**：開放權限，但一定要留紀錄
- **公版設計**：規則固定，要改底層邏輯需另外收費
- **點數使用限額**：用於控制折抵上限，防止虧損
- **2026-04-07 新增**：若有門號更換或點數過戶需求，先以通用型點數修改畫面處理，不急著做專用流程
- **紀錄要求**：點數異動必須保留操作者、原因、正負數值與關聯備註
- **2026-04-10 新增**：等級制先以 3 級公版為主，不開放中間插級；必要時僅保留向上擴充空間
- **單據連動**：修改單據、退貨、補加商品等情境需同步修正點數，並可在異常報表中說明原因
- **測試方式**：需要可快速生成 demo / 測試資料，先驗證規則後再安排客戶 demo
- **2026-05-05 新增 — 退貨退點規則**：
  - **單據商品退貨**：依原銷售單號針對品項退對應點數（如 50 點配件 → 退 50 點，可部分退）
  - **專案退貨**：屬特案（一年數件），需回原銷售單手動調整，不在退貨畫面退
  - 退點操作走既有「點數調整」入口（保留操作者、原因、理由），不在退貨畫面開新按鈕
- **2026-04-27 新增 — 點數扣除三情境**：
  - 銷售退回 → 扣點
  - 單據刪除 → 走「點數回數」（另一條流程）
  - 單據修改 → 重算成正確值
- **2026-04-27 新增 — 進階行銷模組分群**：拆三塊 extension（會員增強 / 點數等級 / LINE 整合），主系統 3,000/月 + 1,000/月每分店；要買就全買
- **2026-04-30 新增 — 專案點數規則**：
  - 採「**可記點 / 不可記點 checkbox**」一刀切（最簡單）
  - 建「保險」商品類別，把保險字樣商品歸入 → 賣保險類別自動給點，後面專案 skip（避免重複算）
  - 規則拉到邏輯層，可由客戶層配置
- **2026-04-30 新增 — 試用流程細節**：
  - 6/1 才開始計費（送一個月試用），中間測試期不算錢
  - 5/9–15 士豪出國，業務話術「負責這塊出國度假」
  - 第一個客戶要建模板（空殼資料庫），下次直接複製
- **2026-04-30 新增 — DOMAIN 處理**：客戶現有官網 DOMAIN 不歸我們管 → 我們申請新 DOMAIN（免費），用 Retarget 過去（不顯示我們網址）
- **2026-04-30 新增 — LIFF 主選單**：零壹做四個（我的點數/我的優惠/客服/盤點）；雨心未來會擴 6 個（加維修進度）
